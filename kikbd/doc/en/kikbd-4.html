4. Personal Configuration

``Personal configuration'' includes all your runtime settings for KiKbd. You should use the KDE Control Center module program kcmikbd for creating/modifing your personal configuration. When you start KiKbd or kcmikbd for the first time it will copy the default system configuration to your personal configuration.

4.1 General settings

Keyboards

Using the Add and Delete buttons you can create/update your personal list of national keyboards. The topmost keyboard ( En - US English Language - in the example on the screenshot shown) becomes the default keyboard when you start KiKbd.

If you use more then two keyboards it is very helpful to check the Use hotlist box. In this case you can switch between the ``hot'' (highlighted) keyboard and the default (first) keyboard by using the Switch key(s).

You can change the choice of ``hot'' keyboard from the KiKbd pop up menu that appears when you right-click on the KiKbd button.

Switch and Alt Switch:

The box on the left shows the key combination chosen for the Switch key .

  • Pressing the Switch key combination switches the keyboard map to the the next one in KiKbd's selected list of keyboard maps (or between the default and the highlighted (``hot'') map, if use hot list is checked).
(You can also change the keyboard map by clicking on the KiKbd button if it is docked in the panel or placed at a corner of the screen.)

The box on the right shows the key chosen as the Alternate Switch key.

  • The ``lower-case'' alternate characters in the keyboard map for the current national keyboard are obtained if the key is typed with the Alternate Switch key held down, and the ``upper-case'' alternate characters are obtained if the key is typed with both the Alternate Switch and the Shift held down.
Each box has a pull-down menu allowing different choices for these special keys to be selected.

  • In the example screenshot, you can switch between keyboards by pressing Right Ctrl and Right Shift keys at the same time (the selected Switch key combination), and you can obtain any alternate symbols defined for the current national keyboard by holding down the Right Alt key (the selected Alternative Switch key) while you type.

Options:

If the Beep box is checked, KiKbd will beep when the keyboard map is changed.

Click on the Advanced button to open the Advanced Options screen.

4.2 Advanced Options

Emulate Caps Lock:

For some keyboards (e.g. Russian, Greek) there are problems with the default CapsLock behavior. Using this option, you instruct KiKbd to emulate CapsLock in order to fix this problem. The Caps Lock state (on or off) is then indicated by the color of the KiKbd language button instead of the corresponding keyboard LED.

World Menu:

You can have a popup menu in any active window by pressing and holding the Switch keys for a sufficiently long time (hold Right Ctrl and Right Shift as in the example screenshot and wait for the popup menu).

Save Classes:

If the Input selection (see below) is ``Class'', you may check the Save Classes box so that the keyboard-map to window-manager (WM) window-class association is automatically saved at the end of each KDE Session, and restored at the beginning of the next session. For example, if you have selected the Russian keyboard in a Kedit window when you log out today, you will get it automatically in your Kedit windows when you log in tomorrow.

Keyboard codes:

The pull-down menu allows you to select the appropriate set of keyboard codes used by your particular keyboard.

Input:

This determines how the keyboard maps are associated with different windows. You can select one of three choices:

  • Global - "standard" behavior: your keyboard map applies to all windows.
  • Window - "extended" behavior: a keyboard map is "remembered" for each top level window, and inherited by each lower-level window created from within it. When the active window changes, the keyboard map will change to the map associated with the active window.
  • Class - "special" behavior: a keyboard map is "remembered" for each window manager (WM) window-class member. For example, if you switch to the Russian keyboard in any window in which the Kedit program is running, you will automatically also get the Russian keyboard in any other Kedit window.

4.3 Style

In the Style screen, You can customize the colors and font used by the KiKbd button that indicates which keyboard map is currently in use, and (by its color) shows whether the emulated Caps Lock is on or if the Alternate Switch key is depressed.

Button Colors:

  • Foreground: Foreground Color used for the text label.
  • Background: Normal Background color used for the text label. ( Not shown in screenshot!.)
  • With CapsLock: Background used for the text label when Caps Lock emulation is selected and the Caps Lock key is activated.
  • With Alternate: Background used for the text label while the Alt Switch key is held down to provide the alternate characters.

Button Font:

Check the Customize Font Check box if you wish to use custom or KDE wide fonts. If this box is checked, clicking the Change Font button will open the Select Font dialog.

4.4 Startup

The Startup screen configures how KiKbd is started.

Autostart:

if the Autostart box is checked, KiKbd will be started automatically at the beginning of each KDE session.

Docked:

If the Docked box is checked, the KiKbd button will be docked into the panel.

Place:

If you don't like docking, and the Docked box is unchecked, you can choose to place the KiKbd button at one the the corners of the display. (The pull-down Place menu presents a list of choices for the location).
