Introduction

Kfind is a frontend for the find utility inspired by the find utility from NT. At this time it is fully functional (except "Open with ..." function) and all in kfind available options are working. You can save, delete, change properties of found files/dirs, store them to archives and ask kfm for opening dir containing this file.

Form version 0.2.9 kfind uses libkde0.7 and is KDE AWARE application.

Installation

Requirements: You have to set the environment variables QTDIR and KDEDIR first. Then you have to install libkde(>=0.7) and kfm(>=0.5.8)!! It is also necessary to patch ktabctl widget in libkde (otherwise there could be sometimes problems by resizing!). Appropriate patch is available in file ktabctl.patch. Finally you can simply type ./configure, make and make install. Then you should copy file $KDEDIR/lib/kfind/config/kfindrc to file ~/.kfindrc (this will enable you to store files to tar archive).

Using kfind

At the program start the most of the menu bar options are disabled, because these actions base are files which are found.

The finding of files and directories is now very easy. You can specify your search in the three dialogs Name&Location, Date Modified and Advanced.

Name&Location: specify the filename and the folders to search in. After specifying the filename you HAVE TO push enter before pushing Find Now button otherwise this filename will not be taken in account by search.

Date Modified: here you can select those files that are created in a special time period

Advanced: a special file type like Postscript-documents, jpegs or Wav -Files. You can also specify the file size here.

The pushbutton Find Now starts the search and beyond the window the results are displayed. The Stop button stops the current search.

When some found file/directory is selected, the items in File menu will be enabled. It is then possible to delete, store to archive or change properties of selected file or call KFM to open folder containing selected file. You can save results of search by invoking menu entry Save Search.

When you doubleclick on file or select Open in File menu kfind will call kfm to open this file with default binding (in case you click on directory it will be opened by kfm)

The filename for storing results is specified in Preferences dialog. This dialog is accessible per Option menu entry. In Preferences dialog you can not only specify filename for result storing. There is possible to look at filetypes a archivers definitions (in future version it will be possible to edit them).

Configuring

As of version 0.2.8 kfind is configurable. It reads from config files information about file for saving search results and about available archives.

Saving configuration: In configuration file is stored filename and format of file used for saving search results. This kfind parameters can be configured using Preferences dialog. This parameter are stored within .kfindrc file in entry [Saving] as follows:

[Saving]
Format=HTML
Filename=/root/result.html

Where Format entry can have values HTML or Plain Text.

Archiver configuration: Archiver parameters are not configurable using Preferences dialog at this time. Archivers available for kfind are specified if .kfindrc entry [Archiver Types] as follows:

[Archiver Types]
Archivers=tar;zip;zoo;

And each archiver is detaily described by its own entry:

[tar]
ExecOnCreate=tar cf %a -C %d %n
ExecOnUpdate=tar uf %a -C %d %n
Comment=Tar

The first line contains command executed when creating new archive and the second one is for updating of existing archive. The third one is comment used for archiver description in Preferences dialog. In commands are used variables %a %f %n %d with following meaning:

%a The complete name of the archive

%f The normal file name

%d The parent directory. If the given argument is a file ( not a directory ) then this will give you the directory this file is in.

%n The name of the file. If the given argument is a directory then this is the name of this directory.
