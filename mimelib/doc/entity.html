NAME

 DwEntity -- Abstract class representing a MIME entity

SYNOPSIS

 
class DwEntity : public DwMessageComponent {

public:

    DwEntity();
    DwEntity(const DwEntity& aEntity);
    DwEntity(const DwString& aStr, DwMessageComponent* aParent=0);
    virtual ~DwEntity();
    const DwEntity& operator = (const DwEntity& aEntity);
    virtual void Parse();
    virtual void Assemble();
    DwHeader& Header() const;
    DwBody& Body() const;

protected:

    DwHeader* mHeader;
    DwBody*   mBody;

#if defined(DEBUG_VERSION)

public:

    virtual void PrintDebugInfo(ostream& aStrm, int aDepth=0) const;
    virtual void CheckInvariants() const;

#endif // defined(DEBUG_VERSION)

};

DESCRIPTION

 RFC-1521 defines an entity as either a message or a body-part, both of which have a header and a body. In the MIME++ class hierarchy, an entity is represented by the class DwEntity. A DwEntity is characterized by the fact that it contains both a DwHeader and a DwBody, just like an entity has a header and a body. These contained objects are the children of a DwEntity object. (See the discussion in the entry for DwMessageComponent.)

Since DwEntity is an abstract base class, you may not instantiate objects of type DwEntity. Instead, you must instantiate objects of a class derived from DwEntity. Just as RFC-1521 specifies that both messages and body-parts are entities, so also MIME++ provides classes DwMessage and DwBodyPart as subclasses of DwEntity.

See also: DwMessageComponent, DwMessage, DwBodyPart

Public Member Functions

DwEntity()
DwEntity(const DwEntity& aEntity)
DwEntity(const DwString& aStr, DwMessageComponent* aParent=0)

The first constructor is the default constructor, which sets the DwEntity object's string representation to the empty string and sets its parent to NULL.

The second constructor is the copy constructor, which copies the string representation from aEntity and all of its children. The parent of the new DwEntity object is set to NULL.

The third constructor copies aStr to the DwEntity object's string representation and sets aParent as its parent. The virtual member function Parse() should be called immediately after this constructor in order to parse the string representation.

virtual ~DwEntity()

const DwEntity& operator = (const DwEntity& aEntity)

This is the assignment operator, which follows regular semantics.

virtual void Parse()

This virtual function, inherited from DwMessageComponent, executes the parse method for DwEntity objects. The parse method parses the string representation of the DwEntity object into its broken-down representation, which consists of a header and a body. The broken-down representation is implemented as contained DwHeader and DwBody objects, which are the children of a DwEntity. (See the discussion in the entry for DwMessageComponent.) After the header and body are parsed, this member function calls the Parse() member function of the contained DwHeader and DwBody objects; you do not have to explicitly parse the DwHeader and DwBody objects.

This member function must be called after the string representation is set or modified, and before the header or {\tt body] of the entity are accessed.

virtual void Assemble()

This virtual function, inherited from DwMessageComponent, executes the assemble method for DwEntity objects. The assemble method assembles the broken-down representation of the DwEntity object into its string representation. The broken-down representation consists of a header and a body, and is implemented as contained DwHeader and DwBody objects, which are the children of a DwEntity. (See the discussion in the entry for DwMessageComponent.) Before the string representation is assembled, this member function will call the Assemble() member function of the contained DwHeader and DwBody objects; you do not have to explicitly assemble the DwHeader and DwBody objects.

This member function should be called after any component of the broken-down representation is set or changed and before the string representation is retrieved.

DwHeader& Header() const

This function returns the DwHeader contained by this object.

DwBody& Body() const

This function returns the DwBody contained by this object.

virtual void PrintDebugInfo(ostream& aStrm, int aDepth=0) const

This virtual function, inherited from DwMessageComponent, prints debugging information about this object to aStrm. It will also call PrintDebugInfo() for any of its child components down to a level of aDepth.

This member function is available only in the debug version of the library.

virtual void CheckInvariants() const

Aborts if one of the invariants of the object fails. Use this member function to track down bugs.

This member function is available only in the debug version of the library.
