DESCRIPTION

 DwHeader represents the header of a message, or a body part in a multipart message, as described in RFC-822 and RFC-1521. A header consists of a series of fields. In MIME++, a DwHeader object contains a list of DwField objects. These contained DwField objects are the children of a DwHeader object. The parent of a DwHeader object is the DwEntity that contains it. (See the discussion or children and parents in the entry for DwMessageComponent.)

DwHeader provides a member function Parse() for parsing it; that is, creating the list of DwField objects corresponding to the fields it contains. It also provides a member function Assemble() for combining its fields to create its string representation.

You can access the field-body for a specific well-known field by using the member function <field>(), where <field> is the field-name of the field with hyphens removed and the first word following a hyphen capitalized. For example, to access the field-body for the ``MIME-version'' field, use MimeVersion(). The member function <field>() will create a field with field-name <field> if such a field does not exist. You can check for the existence of a specific well-known field by using the member function Has<field>(). For example, to check for the existence of the MIME-version field, use HasMimeVersion(). Well-known fields are those documented in RFC-822, RFC-1036, RFC-1521, and possibly other RFCs.

In the case of an extension-field or user-defined-field, you can access the field-body of the field by calling the member function FieldBody() with the field-name as an argument. If the extension-field or user-defined-field does not exist, FieldBody() will create it. You check for the existence of an extension-field or user-defined-field by using the member function HasField() with the field-name as its argument.

You can iterate over all fields in the header by using the member function FirstField() to get the first DwField object in the list, and then using DwField::Next() to iterate through the list.

The first constructor is the default constructor, which sets the DwHeader object's string representation to the empty string and sets its parent to NULL.

The second constructor is the copy constructor, which copies the string representation from aHeader and all of its children. The parent of the new DwHeader object is set to NULL.

The third constructor copies aStr to the DwHeader object's string representation and sets aParent as its parent. The virtual member function Parse() should be called immediately after this constructor in order to parse the string representation. Unless it is NULL, aParent should point to an object of a class derived from DwEntity.

virtual ~DwHeader()

const DwHeader& operator = (const DwHeader& aHeader)

This is the assignment operator, which follows regular semantics.

virtual void Parse()

This virtual function, inherited from DwMessageComponent, executes the parse method for DwHeader objects. The parse method parses the string representation of the DwHeader object into its broken-down representation, which consists of a collection of fields. The broken-down representation is implemented as a list of DwField objects, which are the children of a DwHeader object. (See the discussion in the entry for DwMessageComponent.)

This member function must be called after the string representation is set or modified, and before the fields of the header are accessed.

virtual void Assemble()

This virtual function, inherited from DwMessageComponent, executes the assemble method for DwHeader objects. The assemble method assembles the broken-down representation of the DwHeader object into its string representation. The broken-down representation consists of a collection of fields implemented as a list of DwField objects. Before the string representation is assembled, this member function will call the Assemble() member function of each DwField object in its list; you do not have to explicitly assemble each DwField object.

This member function should be called after any component of the broken-down representation is set or modified and before the string representation is accessed.

virtual DwMessageComponent* Clone() const

This virtual function, inherited from DwMessageComponent, creates a new DwHeader on the free store that has the same value as this DwHeader object. The basic idea is that of a ``virtual copy constructor.''

void AddOrReplaceField(DwField* aField)

Adds a field, implemented as a DwField object, to this header. If a field with the same field-name is already present in the header, it is replaced by the new field.

DwHeader takes responsibility for deleting the added DwField object.

This is an advanced function. Consider using the member functions <field>() (e.g. To(), ContentType(), and so on) and FieldBody() for adding fields to the header.

void AddField(DwField* aField)

Add a field, implemented as a DwField object, to this header. If a field with the same field-name is already present in the header, it is not replaced; thus, duplicate fields may occur when using this member function.

DwHeader takes responsibility for deleting the added DwField object.

This is an advanced function. Consider using the member functions <field>() (e.g. To(), ContentType(), and so on) and FieldBody() for adding fields to the header.

void AddFieldAt(int aPos, DwField* aField)

This member functions follows the semantics of AddField() except that aPos specifies a position for adding the field. A position of 1 indicates the beginning of the list. A position of 0 indicates the end of the list.

This is an advanced function. Consider using the member functions <field>() (e.g. To(), ContentType(), and so on) and FieldBody() for adding fields to the header.

void RemoveField(DwField* aField)

Remove the DwField object from the list. The DwField object is not deleted.

void DeleteAllFields()

Remove all DwField objects from the list and delete them.
