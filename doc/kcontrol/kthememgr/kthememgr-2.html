2. Theme package contents

The Contents screen presents a list of the components contained in the selected Theme package. It also allows you to choose which of these components to install.

Not all possible components are necessarily present in a given Theme package. Components that are present are listed as ``available''; missing components are listed as ``empty''.

The checkbox ``Uninstall parts of previous theme'' determines how missing components are handled when a Theme package is installed:

  • If the box is not checked, those components of the previously-installed Theme which are not supplied by the new Theme package are left in place.
  • If the box is checked, all components of the previous Theme are removed.
If you wish to remove all Theme components by installing the ``Default'' Theme (a themeless Desktop), this box must be checked. (The Default theme has no Theme components.)

Each Theme component also has a checkbox next to it. If the box next to a component listed as ``available'' is checked, that component will be installed when the Apply or OK button is clicked. Components with unchecked boxes will not be installed.

The Clear button clears all the checkboxes for the list of Theme components, and the Invert button exchanges checked and unchecked items.

  • Note: the image files for the Window Titlebar and Window Border components are installed into the directory ~/.kde/share/apps/kwm/pics/ and Panel image files go in ~/.kde/share/apps/kpanel/pics/. If you find you are unable to completely remove a Theme to restore the default desktop, delete the contents of these directories.
