6. KFM Web Browser Configuration: User Agent

In this screen, you can configure the User-Agent String, which is what the KFM Browser reports as its name and version to web servers, when requesting web pages.

This name can be configured to be specific for a given web site.

The usefulness of this is that webservers may decide on what form to send the information (i.e., frames or no frames) depending of which browser application it thinks it is talking to.

Rumor also has it that certain browser distributors' sites have in the past been programmed to send error messages when the the User-Agent String of a competing browser application is reported to them!

The KFM Browser is perfectly capable of handling frames, but this may not be known to a web site that only sends frames to browsers it thinks can handle them. In such a case, you can configure KFM so that for that particular site, it reports the User-Agent String of another well-known browser such as Netscape Navigator (tm), which has user-agent string "Mozilla", plus version information, or Interner Explorer(tm).

The default User-Agent string for KFM is set to Konqueror plus version information (a conqueror of course being the next one in the sequence navigator - explorer - * )!
