2. Features & Tips

  • Disk Navigator can be used to quickly access files in any mounted filesystem, including local files, remote files (eg. NFS mounts), removable media (CD-ROM, Iomega Zip, floppy drive), etc. When a file is selected, the action performed is exactly the same performed when the file is selected in a KFM window.
  • On the top of each menu (except the first one) there is an entry "Open Folder". when this entry is selected, a KFM window is launched using the current folder as URL. If you hold the Shift key when selecting this entry, Disk Navigator opens a terminal in the current folder.
  • Slow devices: accessing a folder on a slow device can stop Disk Navigator for a while (eg. entering a folder with a large number of files on a slow CD-ROM reader). Since you may want to place entry points to slow devices on the root menu, you might accidentally enter such slow entries. To avoid this problem, you can add a "Removable Media" folder to the Global/Local sections, and place links to slow devices in this folder. See the "Options" section to see how to customize the Global/Local sections.
  • Copying ~/.kde and $KDEDIR: the current version of Disk Navigator uses symbolic links to define entry points in the Global/Local sections. Symbolic links are stored in $KDEDIR/share/disknav (Global) and ~/.kde/share/disknav (Local). Therefore, if you want to copy a KDE directories (for instance  /.kde to create a backup copy of your KDE settings), you should use cp with the -d option. This way, symbolic links are copied as symbolic links. If, for instance, you use: "cp -r ~/.kde kde-backup" (without the -d option) cp will follow symbolic links in the disknav directory.
